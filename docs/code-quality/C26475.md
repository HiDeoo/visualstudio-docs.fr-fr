# <a name="c26475-no_function_style_casts"></a>C26475 NO_FUNCTION_STYLE_CASTS

« N’utilisez pas les casts de style de fonction. »

**C++ Instructions principales**: ES. 49 : Si vous devez utiliser un cast, utilisez un cast nommé

Les casts de style fonction (par exemple `int(1.1)`,) sont une autre incarnation de casts de style `(int)1.1`C (comme), qui ont une sécurité douteuse. Plus précisément, le compilateur n’essaie pas de vérifier si une perte de données peut se produire dans les casts en C ou dans les conversions de fonction. Dans les deux cas, il est préférable d’éviter le cast ou d’utiliser l’initialiseur d’accolades, si possible. Si aucun de ces éléments ne fonctionne, les casts statiques peuvent être adaptés, mais il est toujours préférable d’utiliser les utilitaires de la bibliothèque de prise en charge des instructions :

- GSL :: Narrow garantit la conversion sans perte et provoque un blocage au moment de l’exécution s’il n’est pas possible.
- GSL :: narrow_cast indique clairement que la conversion peut perdre des données et qu’elle est acceptable.

## <a name="remarks"></a>Notes

- Cette règle se déclenche uniquement pour les constantes de types primitifs. Il s’agit des cas où le compilateur peut détecter clairement la perte de données et émettre une erreur si l’initialiseur d’accolade est utilisé. Les cas nécessitant une exécution au moment de l’exécution sont marqués par C26493 NO_CSTYLE_CAST.
- Les initialiseurs par défaut ne sont pas signalés `int()`(par exemple).

## <a name="example"></a>Exemple

conversion dangereuse

```cpp
constexpr auto planck_constant = float( 6.62607004082e-34 ); // C26475
```

```cpp
dangerous conversion – detecting potential data loss
constexpr auto planck_constant = float{ 6.62607004082e-34 }; // Error C2397
```

```cpp
dangerous conversion – corrected
constexpr auto planck_constant = double{ 6.62607004082e-34 };
```
