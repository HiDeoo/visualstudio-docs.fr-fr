# <a name="c6217"></a>C6217

avertissement C6217 : Cast implicite entre types d’entiers sémantiquement différents : test de HRESULT avec’not'. Utilisez la macro [Succeeded](/windows/desktop/api/winerror/nf-winerror-succeeded) ou [failed](/windows/desktop/api/winerror/nf-winerror-failed) à la place.

Cet avertissement indique qu’une `HRESULT` est testée à l’aide de l’opérateur NOT (`!`). Une réussite (`S_OK`) dans `HRESULT` est indiquée par une valeur de 0. Toutefois, 0 indique un échec pour un type booléen. Le test de `HRESULT` avec l’opérateur NOT (`!`) pour déterminer quel bloc de code exécuter peut provoquer la suite du chemin de code incorrect. Cela entraînera des résultats indésirables.

## <a name="example"></a>Exemple

Le code suivant génère cet avertissement, car l’opérateur NOT est utilisé pour déterminer la réussite ou l’échec d’une valeur `HRESULT`. Dans ce cas, le chemin de code incorrect est exécuté, car `( !hr )` exécute le code d’échec :

```cpp
#include <windows.h>
#include <objbase.h>

void f( )
{
  HRESULT hr = CoInitialize(NULL);
  if (!hr)
  {
    // failure code ...
  }
  else
  {
    // success code ...
  }
}
```

Pour corriger cet avertissement, le code suivant utilise la macro `FAILED` pour Rechercher l’échec :

```cpp
#include <windows.h>
#include <objbase.h>

void f( )
{
  HRESULT hr = CoInitialize(NULL);
  if (FAILED(hr))
  {
    // failure code ...
  }
  else
  {
    // success code ...
  }
}
```

Pour cet avertissement, le type `SCODE` équivaut à `HRESULT`.

La valeur de succès typique de HRESULT (`S_OK`) est `false` lorsqu’elle est testée en tant que valeur booléenne.

Pour vérifier si `HRESULT` est un succès, utilisez la macro `SUCCEEDED` à la place.
