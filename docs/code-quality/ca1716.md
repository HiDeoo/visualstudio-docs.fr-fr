# <a name="ca1716-identifiers-should-not-match-keywords"></a>CA1716 : Les identificateurs ne doivent pas correspondre à des mots clés

|||
|-|-|
|TypeName|IdentifiersShouldNotMatchKeywords|
|CheckId|CA1716|
|Category|Microsoft.Naming (en)|
|Modification avec rupture|Rupture|

## <a name="cause"></a>Cause

Le nom d’un membre de namespace, de type ou d’interface correspond à un mot clé réservé dans un langage de programmation.

Par défaut, cette règle ne regarde que les espaces de noms, les types et les membres visibles à l’extérieur, mais vous pouvez [configurer la visibilité](#api-surface) et [les types de symboles.](#analyzed-symbol-kinds)

## <a name="rule-description"></a>Description de la règle

Les identifiants pour les espaces de nom, les types et les membres virtuels et d’interface ne doivent pas correspondre aux mots clés qui sont définis par des langues qui ciblent l’heure courante de la langue commune. Selon la langue utilisée et le mot clé, les erreurs et les ambiguïtés des compilateurs peuvent rendre la bibliothèque difficile à utiliser.

Cette règle vérifie les mots clés dans les langues suivantes :

- Visual Basic
- C#
- C++/CLI

La comparaison insensible aux cas est utilisée pour les mots clés visualistiques de base, et la comparaison sensible aux cas est utilisée pour les autres langues.

## <a name="how-to-fix-violations"></a>Comment corriger les violations

Sélectionnez un nom qui n’apparaît pas dans la liste des mots clés.

## <a name="when-to-suppress-warnings"></a>Quand supprimer les avertissements

Vous pouvez supprimer un avertissement de cette règle si vous êtes convaincu que l’identifiant ne confondra pas les utilisateurs de l’API, et que la bibliothèque est utilisable dans toutes les langues disponibles en .NET.

## <a name="configurability"></a>Configurabilité

### <a name="api-surface"></a>Surface Api

Si vous exécutez cette règle à partir [d’analyseurs FxCop](install-fxcop-analyzers.md) (et non avec l’analyse héritée), vous pouvez configurer les parties de votre base de code pour exécuter cette règle, en fonction de leur accessibilité. Par exemple, pour spécifier que la règle ne doit s’exécuter que par rapport à la surface non publique de l’API, ajoutez la paire de valeur clé suivante à un fichier .editorconfig dans votre projet :

```ini
dotnet_code_quality.ca1716.api_surface = private, internal
```

Vous pouvez configurer cette option pour cette seule règle, pour toutes les règles, ou pour toutes les règles de cette catégorie (Naming). Pour plus d’informations, voir [Configurer les analyseurs FxCop](configure-fxcop-analyzers.md).

### <a name="analyzed-symbol-kinds"></a>Types de symboles analysés

Vous pouvez configurer les types de membres qui seront analysés par cette règle. Utilisez une ou plusieurs `Namespace` `NamedType`de `Method` `Property`ces `Event` `Parameter` valeurs, , , , , comme une liste séparée de virgule. Notez que la valeur par défaut comprend tous les membres ci-dessus.

```ini
dotnet_code_quality.CA1716.analyzed_symbol_kinds = Namespace, NamedType, Method, Property, Event
```
