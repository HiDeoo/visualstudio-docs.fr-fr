# <a name="c6283"></a>C6283
AVERTISSEMENT C6283 : \<variable > est alloué avec un tableau New [], mais supprimé avec une variable scalaire Delete

 Cet avertissement apparaît uniquement dans C++ le code et indique que la fonction d’appel a alloué de manière incohérente la mémoire avec le tableau @no__t opérateur-1, mais l’a libéré avec l’opérateur scalaire `delete`. Ce défaut peut provoquer des fuites, des altérations de la mémoire et, dans les situations où les opérateurs ont été remplacés, se bloque. Si la mémoire est allouée avec le tableau `new []`, elle doit généralement être libérée avec le tableau `delete[]`.

## <a name="example"></a>Exemple
 Le code suivant génère cet avertissement :

```
void f( )
{
  char *str = new char[50];
  // code ...
  delete str;
}
```

 Pour corriger cet avertissement, utilisez le code suivant :

```
void f( )
{
  char *str = new char[50];
  // code ...
  delete[] str;
}
```

 Warning C6283 s’applique uniquement aux tableaux de types primitifs tels que, des entiers ou des caractères. Si les éléments du tableau sont des objets de type classe, l’avertissement [C6278](../code-quality/c6278.md) est émis.

 L’utilisation des nouvelles et des suppressions présente de nombreux pièges en termes de fuites de mémoire et d’exceptions. Pour éviter l’ensemble de ces types de fuites et de problèmes d’exception, utilisez les mécanismes fournis par C++ la bibliothèque STL (Standard Template Library). Il s’agit notamment de [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)et [Vector](/cpp/standard-library/vector). Pour plus d’informations, consultez [pointeurs intelligents](/cpp/cpp/smart-pointers-modern-cpp) et [ C++ bibliothèque standard](/cpp/standard-library/cpp-standard-library-reference).
