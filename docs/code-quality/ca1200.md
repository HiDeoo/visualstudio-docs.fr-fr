# <a name="ca1200-avoid-using-cref-tags-with-a-prefix"></a>CA1200 : Éviter d’utiliser des balises cref avec un préfixe

|||
|-|-|
|TypeName|AvoidUsingCrefTagsWithAPrefix|
|CheckId|CA1200|
|Category|Microsoft. documentation|
|Modification avec rupture|Sans rupture|

## <a name="cause"></a>Cause

La balise [cref](https://docs.microsoft.com/dotnet/csharp/programming-guide/xmldoc/cref-attribute) dans un commentaire de documentation XML utilise un [préfixe](https://docs.microsoft.com/dotnet/csharp/programming-guide/xmldoc/processing-the-xml-file).

## <a name="rule-description"></a>Description de la règle

L’attribut [cref](https://docs.microsoft.com/dotnet/csharp/programming-guide/xmldoc/cref-attribute) dans une balise de documentation XML signifie « référence du code ». Il indique que le texte interne de la balise est un élément de code tel qu’un type, une méthode ou une propriété. Évitez d' `cref` utiliser des balises avec des préfixes, car cela empêche le compilateur de vérifier les références. Cela empêche également l’environnement de développement intégré (IDE) de Visual Studio de rechercher et de mettre à jour ces références de symboles lors des refactorisations. Il est recommandé d’utiliser la syntaxe complète sans préfixes pour référencer les noms de symboles dans les balises cref.

> [!NOTE]
> La règle ca1200 n’est pas disponible dans l’analyse héritée. Il a été introduit pour la première fois dans les [analyseurs FxCop](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers).

## <a name="how-to-fix-violations"></a>Comment corriger les violations

Pour corriger une violation de cette règle, supprimez le préfixe `cref` de la balise. Par exemple, les deux extraits de code suivants illustrent une violation de la règle et comment la corriger :

```csharp
// Violates CA1200
/// <summary>
/// Type <see cref="T:C" /> contains method <see cref="F:C.F" />
/// </summary>
class C
{
    public void F() { }
}
```

```csharp
// Does not violate CA1200
/// <summary>
/// Type <see cref="C" /> contains method <see cref="F" />
/// </summary>
class C
{
    public void F() { }
}
```

## <a name="when-to-suppress-warnings"></a>Quand supprimer les avertissements

Il est recommandé de supprimer cet avertissement si la référence de code doit utiliser un préfixe, car le type référencé n’est pas localisable par le compilateur. Par exemple, si une référence de code fait référence à un attribut spécial dans l’infrastructure complète, mais que le fichier est compilé par rapport à l’infrastructure portable, vous pouvez supprimer cet avertissement.
