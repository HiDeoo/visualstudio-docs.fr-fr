# <a name="c26448-use_gsl_finally"></a>C26448 USE_GSL_FINALLY

Envisagez d’utiliser `gsl::finally` si l’action finale est prévue.

C++Instructions principales : [GSL. util : Utilitaires @ no__t-0

La bibliothèque de prise en charge des instructions fournit un utilitaire pratique pour implémenter le concept d' *action final* . Étant donné C++ que le langage ne prend pas en charge les constructions **try-finally** , il est devenu courant d’implémenter des types de nettoyage personnalisés qui invoqueront des actions arbitraires en cas de destruction. L’utilitaire `gsl::finally` est implémenté de cette façon et offre un moyen plus uniforme d’effectuer des actions finales dans une base de code.

Il existe également des cas où les actions finales sont exécutées dans un style C à l’aide d’instructions **goto** (ce qui est généralement déconseillé par [C26438 NO_GOTO](c26438.md)). Il est difficile de détecter l’intention exacte du code qui utilise fortement **goto**, mais certains heuristiques peuvent vous aider à trouver de meilleurs candidats au nettoyage.

## <a name="remarks"></a>Notes

- Cette règle est très légère et utilise des noms d’étiquette pour deviner les opportunités d’utilisation des objets d’action finaux.
- Les noms d’étiquette qui peuvent déclencher un avertissement contiennent des mots tels que « end », « final », « Clean », et ainsi de suite.
- Les avertissements s’affichent dans les instructions **goto** . Vous pouvez voir des résultats détaillés dans certains cas, mais cela peut vous aider à hiérarchiser le code en fonction de sa complexité.
- Cette règle est toujours en paire avec [C26438 NO_GOTO](c26438.md). Selon les priorités, l’une d’entre elles peut être désactivée.

## <a name="example-cleanup-with-multiple-goto-statements"></a>Exemple : Nettoyage avec plusieurs instructions goto

```cpp
void poll(connection_info info)
{
    connection c = {};
    if (!c.open(info))
        return;

    while (c.wait())
    {
        connection::header h{};
        connection::signature s{};
        if (!c.read_header(h))
            goto end;               // C26448 and C26438
        if (!c.read_signature(s))
            goto end;               // C26448 and C26438
        // ...
    }

end:
    c.close();
}
```

## <a name="example-cleanup-with-multiple-goto-statements-replaced-by-gslfinally"></a>Exemple : Nettoyage avec plusieurs instructions goto remplacées par GSL :: finally

```cpp
void poll(connection_info info)
{
    connection c = {};
    if (!c.open(info))
        return;

    auto end = gsl::finally([&c] { c.close(); });
    while (c.wait())
    {
        connection::header h{};
        connection::signature s{};
        if (!c.read_header(h))
            return;
        if (!c.read_signature(s))
            return;
        // ...
    }
}
```
