# <a name="c6216"></a>C6216
AVERTISSEMENT C6216 : cast inséré par le compilateur entre des types intégraux sémantiquement différents : type booléen vers HRESULT

 Cet avertissement indique qu’une valeur booléenne est utilisée en tant que `HRESULT` sans cast explicite. Les types booléens indiquent la réussite d’une valeur différente de zéro ; la réussite (`S_OK`) dans `HRESULT` est indiquée par la valeur 0. La valeur d’échec classique pour les fonctions qui retournent une valeur booléenne false est un état de réussite lorsqu’elle est testée en tant que `HRESULT`. Cela risque d’entraîner des résultats incorrects.

## <a name="example"></a>Exemple
 Le code suivant génère cet avertissement :

```
#include <windows.h>
BOOL IsEqual(REFGUID, REFGUID);

HRESULT f( REFGUID riid1, REFGUID riid2 )
{
  // code ...
  return IsEqual(riid1, riid2);
}
```

 Pour corriger cet avertissement, utilisez le code suivant :

```
#include <windows.h>
BOOL IsEqual(REFGUID, REFGUID);

HRESULT f( REFGUID riid1, REFGUID riid2 )
{
  if (IsEqual(riid1, riid2) == TRUE)
  {
    // code ...
    return S_OK;
  }
  else
  {
    // code ...
    return E_FAIL;
  }
}
```

 Pour cet avertissement, le type `SCODE` équivaut à `HRESULT`.

 Pour plus d’informations, consultez [macro réussie](/windows/win32/api/winerror/nf-winerror-succeeded) et échec de la [macro](/windows/win32/api/winerror/nf-winerror-failed).
