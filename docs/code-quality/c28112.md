# <a name="c28112"></a>C28112

AVERTISSEMENT C28112 : Une variable accessible via une fonction Interlocked doit toujours être accessible via une fonction verrouillée

Voir ligne *[numéro]* : Il n’est pas toujours possible d’accéder à une variable qui est accessible via la famille de fonctions « no__t-0 », de toute autre manière.

Une variable qui est accessible à l’aide des routines de prise en charge d’Executive, telles que InterlockedCompareExchangeAcquire, est accessible par la suite à l’aide d’une fonction différente. Bien que certaines affectations, accès et comparaisons ordinaires à des variables utilisées par les routines @ no__t-0 bloquées puissent être accessibles en toute sécurité à l’aide d’une fonction différente, le risque est suffisamment grand pour justifier l’examen de chaque instance.

## <a name="example"></a>Exemple

L'exemple de code suivant génère cet avertissement :

```cpp
inter_var --;
...
InterlockedIncrement(&inter_var);
```

L'exemple de code suivant évite cet avertissement :

```cpp
InterlockedDecrement(&inter_var);
...
InterlockedIncrement(&inter_var);
```
