# <a name="ca1034-nested-types-should-not-be-visible"></a>CA1034 : Les types imbriqués ne doivent pas être visibles

|||
|-|-|
|TypeName|NestedTypesShouldNotBeVisible|
|CheckId|CA1034|
|Category|Microsoft.Design (en)|
|Modification avec rupture|Rupture|

## <a name="cause"></a>Cause

Un type visible à l’extérieur contient une déclaration de type extérieurement visible. Les énumérations imbriquées, les types protégés et les modèles de constructeurs sont exemptés de cette règle.

## <a name="rule-description"></a>Description de la règle
Un type imbriqué est un type déclaré dans le champ d’application d’un autre type. Les types imbriqués sont utiles pour encapsulant les détails de mise en œuvre privés du type contenant. Utilisés à cette fin, les types imbriqués ne doivent pas être visibles de l'extérieur.

N’utilisez pas les types imbriqués visibles à l’extérieur pour le regroupement logique ou pour éviter les collisions de noms; au lieu de cela, utilisez des espaces nominaux.

Les types imbriqués comprennent la notion d’accessibilité des membres, que certains programmeurs ne comprennent pas clairement.

Les types protégés peuvent être utilisés dans les sous-classes et les types imbriqués dans les scénarios de personnalisation à l’avance.

## <a name="how-to-fix-violations"></a>Comment corriger les violations
Si vous n’avez pas l’intention que le type imbriqué soit visible à l’extérieur, modifiez l’accessibilité du type. Sinon, retirez le type imbriqué de son parent. Si le but de la nidification est de catégoriser le type imbriqué, utilisez un espace nom pour créer la hiérarchie à la place.

## <a name="when-to-suppress-warnings"></a>Quand supprimer les avertissements
Ne supprimez aucun avertissement de cette règle.
