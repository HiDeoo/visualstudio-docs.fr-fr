# <a name="ca1066-implement-iequatable-when-overriding-equals"></a>CA1066 : Implémenter IEquatable au moment de remplacer Equals

|||
|-|-|
|TypeName|EquatableAnalyzer|
|CheckId|Ca1066|
|Category|Microsoft. Design|
|Modification avec rupture|Sans rupture|

## <a name="cause"></a>Cause

Un type valeur (struct) substitue la <xref:System.Object.Equals%2A> méthode, mais n’implémente <xref:System.IEquatable%601>pas.

## <a name="rule-description"></a>Description de la règle

Une méthode de substitution <xref:System.Object.Equals%2A> de type valeur indique qu’elle prend en charge la comparaison de deux instances du type pour déterminer l’égalité des valeurs. Envisagez <xref:System.IEquatable%601> d’implémenter l’interface pour prendre en charge des tests fortement typés pour vérifier l’égalité. Cela garantit que les appelants effectuant des vérifications d’égalité <xref:System.IEquatable%601.Equals%2A?displayProperty=fullName> appellent la méthode fortement typée et évitent la conversion boxing de l’argument, ce qui améliore les performances. Vous pourrez trouver plus d’informations [ici](/dotnet/api/system.iequatable-1#notes-to-implementers).

Votre <xref:System.IEquatable%601.Equals%2A?displayProperty=fullName> implémentation de doit retourner des résultats cohérents <xref:System.Object.Equals%2A>avec.

## <a name="how-to-fix-violations"></a>Comment corriger les violations

Pour corriger une violation, implémentez <xref:System.IEquatable%601> et <xref:System.Object.Equals%2A> mettez à jour la substitution pour appeler cette méthode implémentée. Par exemple, les deux extraits de code suivants illustrent une violation de la règle et comment la corriger :

```csharp
public struct S
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public override int GetHashCode()
        => _value.GetHashCode();

    public override bool Equals(object other)
        => other is S otherS && _value == otherS._value;
}
```

```csharp
using System;

public struct S : IEquatable<S>
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public override int GetHashCode()
        => _value.GetHashCode();

    public override bool Equals(object other)
        => other is S otherS && Equals(otherS);

    public bool Equals(S other)
        => _value == other._value;
}
```

## <a name="when-to-suppress-warnings"></a>Quand supprimer les avertissements

Il convient de supprimer les violations de cette règle si la conception et les performances de l’implémentation de l’interface ne sont pas critiques.
