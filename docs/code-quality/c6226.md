# <a name="c6226"></a>C6226

AVERTISSEMENT C6226 : Cast implicite entre types d’entiers sémantiquement différents : assignation de-1 à HRESULT. Utilisez à la place E_FAIL.

Cet avertissement indique qu’une `HRESULT` est assignée ou initialisée à une valeur explicite de-1. Cet avertissement est souvent dû à une confusion accidentelle entre les types Integer et `HRESULT`. Pour indiquer la réussite, utilisez la constante symbolique `S_OK` à la place. Pour indiquer un échec, utilisez les constantes symboliques qui commencent par la*constante*E_, par exemple `E_FAIL`.

Pour plus d’informations, consultez les macros ayant [réussi](/windows/desktop/api/winerror/nf-winerror-succeeded) et [échoué](/windows/desktop/api/winerror/nf-winerror-failed) .

## <a name="example"></a>Exemple

Le code suivant génère cet avertissement :

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (FAILED(CoGetMalloc(1, &pMalloc)))
  {
    hr = -1;
    // code ...
  }
  else
  {
    // code ...
  }
}
```

Pour corriger cet avertissement, utilisez le code suivant :

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (FAILED(CoGetMalloc(1, &pMalloc)))
  {
    hr = E_FAIL;
    // code ...
  }
  else
  {
    // code ...
  }
}
```

Pour cet avertissement, le type `SCODE` est traité comme une `HRESULT`.

L’utilisation de `malloc` et `free` (et les API de mémoire dynamique associées) présente de nombreux pièges comme une cause de fuites de mémoire et d’exceptions. Pour éviter ces types de fuites et de problèmes d’exception, utilisez les classes de pointeur et de C++ conteneur fournies par la bibliothèque standard. Il s’agit notamment de [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)et [Vector](/cpp/standard-library/vector). Pour plus d’informations, consultez [pointeurs intelligents](/cpp/cpp/smart-pointers-modern-cpp) et [ C++ bibliothèque standard](/cpp/standard-library/cpp-standard-library-reference).
