# <a name="c6215"></a>C6215
AVERTISSEMENT C6215 : cast entre types d’entiers sémantiquement différents : type booléen vers HRESULT

 Cet avertissement indique qu’une valeur booléenne est castée en un `HRESULT`. Les types booléens indiquent la réussite d’une valeur différente de zéro, tandis que la réussite (`S_OK`) dans `HRESULT` est indiquée par la valeur 0. Effectuer un cast d’un type booléen en un `HRESULT`, puis l’utiliser dans une expression de test produit un résultat incorrect.

 Cet avertissement se produit fréquemment lorsqu’un booléen est utilisé en tant qu’argument pour une macro `SUCCEEDED` ou `FAILED`, qui convertit explicitement leurs arguments en un `HRESULT`.

## <a name="example"></a>Exemple
 Le code suivant génère cet avertissement :

```
#include <windows.h>
BOOL IsEqual(REFGUID, REFGUID);

void f( REFGUID riid1, REFGUID riid2 )
{
  if (SUCCEEDED( IsEqual( riid1, riid2 ) )) //warning 6215
  {
    // success code ...
  }
  else
  {
    // failure code ...
  }
}
```

 En règle générale, les macros `SUCCEEDED` ou `FAILED` doivent être appliquées uniquement à `HRESULT`.

 Pour corriger cet avertissement, utilisez le code suivant :

```
#include <windows.h>
BOOL IsEqual(REFGUID, REFGUID);

void f( REFGUID riid1, REFGUID riid2 )
{
  if (IsEqual( riid1, riid2 ) == TRUE)
  {
    // code for riid1 == riid2
  }
  else
  {
    // code for riid1 != riid2
  }
}
```

 Pour plus d’informations, consultez la [macro SUCCEEDED](/windows/win32/api/winerror/nf-winerror-succeeded) et l’échec de la [macro](/windows/win32/api/winerror/nf-winerror-failed)
