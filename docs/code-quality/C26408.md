# <a name="c26408-no_malloc_free"></a>C26408 NO_MALLOC_FREE
Cet avertissement indique où `malloc` ou `free` est appelé explicitement conformément à R. 10 : Évitez les `malloc` et `free`. Un correctif potentiel pour ces avertissements serait d’utiliser [std :: make_unique](/cpp/standard-library/memory-functions#make_unique) pour éviter la création et la destruction explicites d’objets. Si ce correctif n’est pas acceptable, l’opérateur [New et Delete](/cpp/cpp/new-and-delete-operators) doit être préféré. Dans certains cas, si les exceptions ne sont pas les bienvenues, `malloc` et `free` peuvent être remplacés par la version nothrow des opérateurs `new` et `delete`.

## <a name="remarks"></a>Notes
- Pour détecter malloc (), nous vérifions si un appel appelle une fonction globale portant le nom « malloc » ou « std :: malloc ». La fonction doit retourner un pointeur vers `void` et accepter un paramètre de type intégral non signé.
- Pour détecter Free (), nous vérifions les fonctions globales avec les noms « Free » ou « std :: Free » qui ne retournent aucun résultat et acceptent un paramètre, qui est un pointeur vers `void`.
