# <a name="ca1827-do-not-use-countlongcount-when-any-can-be-used"></a>CA1827 : n’utilisez pas Count/LongCount lorsqu’un peut être utilisé

|||
|-|-|
|TypeName|DoNotUseCountWhenAnyCanBeUsedAnalyzer|
|CheckId|CA1827|
|Category|Microsoft. performance|
|Modification avec rupture|Sans rupture|

## <a name="cause"></a>Cause

La <xref:System.Linq.Enumerable.Count%2A> méthode <xref:System.Linq.Enumerable.LongCount%2A> ou a été utilisée <xref:System.Linq.Enumerable.Any%2A> là où la méthode serait plus efficace.

## <a name="rule-description"></a>Description de la règle

Cette règle signale <xref:System.Linq.Enumerable.Count%2A> les <xref:System.Linq.Enumerable.LongCount%2A> appels de méthode et LINQ utilisés pour vérifier si la collection a au moins un élément. Ces appels de méthode requièrent l’énumération de la collection entière pour calculer le nombre. La même vérification est plus rapide avec <xref:System.Linq.Enumerable.Any%2A> la méthode, car elle évite l’énumération de la collection.

> [!NOTE]
> La règle CA1827 n’est pas disponible dans l’analyse héritée. Il a été introduit pour la première fois dans les [analyseurs FxCop](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers) version 2.9.6.

## <a name="how-to-fix-violations"></a>Comment corriger les violations

Pour corriger une violation, remplacez l' <xref:System.Linq.Enumerable.Count%2A> appel <xref:System.Linq.Enumerable.LongCount%2A> de la méthode ou <xref:System.Linq.Enumerable.Any%2A> par la méthode. Par exemple, les deux extraits de code suivants illustrent une violation de la règle et comment la corriger :

```csharp
using System.Collections.Generic;
using System.Linq;

class C
{
    public string M1(IEnumerable<string> list)
        => list.Count() != 0 ? "Not empty" : "Empty";

    public string M2(IEnumerable<string> list)
        => list.LongCount() > 0 ? "Not empty" : "Empty";
}
```


```csharp
using System.Collections.Generic;
using System.Linq;

class C
{
    public string M1(IEnumerable<string> list)
        => list.Any() ? "Not empty" : "Empty";

    public string M2(IEnumerable<string> list)
        => list.Any() ? "Not empty" : "Empty";
}
```

> [!TIP]
> Un correctif de code est disponible pour cette règle dans Visual Studio. Pour l’utiliser, placez le curseur sur la violation du tableau, puis appuyez sur **CTRL**+**.** (point). Sélectionnez **ne pas utiliser Count () ou LongCount () si any () peut être utilisé** à partir de la liste d’options présentée.
>
> ![Correction de code pour CA1827-n’utilisez pas Count () ou LongCount () quand any () peut être utilisé](media/ca1827-codefix.png)

## <a name="when-to-suppress-warnings"></a>Quand supprimer les avertissements

Il est possible de supprimer en toute sécurité une violation de cette règle si vous ne vous inquiétez pas de l’impact sur les performances d’une énumération de collection inutile pour calculer le nombre.
