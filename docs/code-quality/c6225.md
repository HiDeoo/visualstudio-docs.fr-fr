# <a name="c6225"></a>C6225

AVERTISSEMENT C6225 : cast implicite entre types d’entiers sémantiquement différents : assignation de 1 ou TRUE à HRESULT. Utilisez S_FALSE à la place

Cet avertissement indique qu’une `HRESULT` est assignée ou initialisée avec une valeur d’un 1 explicite. Les types booléens indiquent la réussite d’une valeur différente de zéro ; la réussite (`S_OK`) dans `HRESULT` est indiquée par la valeur 0. Cet avertissement est souvent dû à une confusion accidentelle entre les types booléen et `HRESULT`. Pour indiquer la réussite de l’opération, la constante symbolique `S_OK` doit être utilisée.

## <a name="example"></a>Exemple

Dans le code suivant, l’affectation de `HRESULT` génère cet avertissement :

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (SUCCEEDED(CoGetMalloc(1, &pMalloc)))
  {
    // code ...
    hr = S_OK;
  }
  else
  {
    // code ...
    hr = 1;
  }
}
```

Pour corriger cet avertissement, utilisez le code suivant :

```cpp
VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (SUCCEEDED(CoGetMalloc(1, &pMalloc)))
  {
    hr = S_OK;
    // code ...
  }
  else
  {
    hr = S_FALSE;
    // code ...
  }
}
```

Pour cet avertissement, le type `SCODE` équivaut à `HRESULT`.

Pour indiquer un échec, `E_FAIL`, ou une autre constante, doit être utilisé.
