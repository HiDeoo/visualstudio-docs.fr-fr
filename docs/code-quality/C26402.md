# <a name="c26402dont_heap_allocate_movable_result"></a>C26402 DONT_HEAP_ALLOCATE_MOVABLE_RESULT

Pour éviter toute confusion quant au fait qu’un pointeur possède un objet, une fonction qui retourne un objet mobile doit l’allouer sur la pile et la retourner par valeur au lieu de retourner un objet alloué par tas. Si la sémantique du pointeur est requise, retourne un pointeur intelligent au lieu d’un pointeur brut. Consultez [ C++ instructions principales R. 3](https://github.com/isocpp/CppCoreGuidelines/blob/master/CppCoreGuidelines.md#Rr-ptr): *Warn si une fonction retourne un objet qui a été alloué dans la fonction, mais qui a un constructeur de déplacement. Suggérez de le retourner par valeur à la place.*
