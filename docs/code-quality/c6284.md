# <a name="c6284"></a>C6284
AVERTISSEMENT C6284 : objet passé en tant que paramètre'% d’quand une chaîne est requise dans l’appel à \<function >.

 Cet avertissement indique que la chaîne de format spécifie une chaîne, par exemple, une spécification `%s` pour `printf` ou `scanf`, C++ mais qu’un objet a été passé à la place.

 Ce défaut peut produire une sortie ou des blocages incorrects.

 Ce message est souvent signalé en raison du passage C++ d’un objet qui implémente un type de chaîne, par exemple, `std::string`, `CComBSTR` ou `bstr_t`, dans un appel de style C `printf`. En fonction de l’implémentation de la C++ classe, autrement dit, si les opérateurs de cast appropriés sont définis C++ , les objets String peuvent souvent être utilisés de manière transparente à chaque fois que des chaînes C sont requises ; Toutefois, étant donné que les paramètres des fonctions de style `printf` sont essentiellement non typés, aucune conversion en chaîne ne se produit.

 En fonction de l’objet, il peut être approprié d’insérer un opérateur `static_cast` dans le type de chaîne approprié, par exemple, `char *` ou `TCHAR``*`, ou d’appeler une fonction membre qui retourne une chaîne, par exemple, `c_str()`, sur les instances de `std::string`.

## <a name="example"></a>Exemple
 Le code suivant génère cet avertissement, car un `CComBSTR` est passé à la fonction `sprintf` :

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf(buff,"%ws",bstrValue);
}
```

 Le code suivant utilise un cast statique pour corriger cet avertissement :

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf_s(buff,50,"%ws",static_cast<wchar_t *>(bstrValue));
}
```
