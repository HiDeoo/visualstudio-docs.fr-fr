# <a name="ca1506-avoid-excessive-class-coupling"></a>CA1506 : Éviter les couplages de classe excessifs

|||
|-|-|
|TypeName|AvoidExcessiveClassCoupling|
|CheckId|CA1506|
|Category|Microsoft.Maintainability (en)|
|Modification avec rupture|Rupture|

## <a name="cause"></a>Cause

Un type ou une méthode est couplé avec de nombreux autres types. Les types générés par le compilateur sont exclus de cette mesure.

## <a name="rule-description"></a>Description de la règle

Cette règle mesure l'accouplement de classes en comptant le nombre de références de type uniques contenues dans un type ou une méthode.

Les types et les méthodes qui ont un degré élevé de couplage de classe peuvent être difficiles à maintenir. C’est une bonne pratique d’avoir des types et des méthodes qui présentent un faible accouplement et une grande cohésion.

## <a name="how-to-fix-violations"></a>Comment corriger les violations

Pour corriger cette violation, essayez de redessiner le type ou la méthode pour réduire le nombre de types auxquels il est couplé.

## <a name="when-to-suppress-warnings"></a>Quand supprimer les avertissements

Exclure cet avertissement lorsque le type ou la méthode est considéré comme maintenable malgré son grand nombre de dépendances sur d’autres types.
