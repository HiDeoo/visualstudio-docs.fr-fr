# <a name="ca1509-invalid-entry-in-code-metrics-configuration-file"></a>Ca1509 : entrée non valide dans le fichier de configuration des métriques du code

|||
|-|-|
|TypeName|CodeMetricsAnalyzer|
|CheckId|Ca1509|
|Category|Microsoft. maintenabilité|
|Modification avec rupture|Sans rupture|

## <a name="cause"></a>Cause

Les règles de métrique du code, telles que [CA1501](ca1501.md), [CA1502](ca1502.md), [ca1505](ca1505.md) et [CA1506](ca1506.md), ont fourni un `CodeMetricsConfig.txt` fichier de configuration nommé qui a une entrée non valide.

## <a name="rule-description"></a>Description de la règle

L’implémentation des [analyseurs FxCop](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers) des règles d’analyse de la [métrique du code](code-metrics-values.md) permet aux utilisateurs finaux `CodeMetricsConfig.txt`de fournir un [fichier supplémentaire](https://github.com/dotnet/roslyn/blob/release/dev16.6/docs/analyzers/Using%20Additional%20Files.md) nommé. Ce fichier contient des entrées permettant de configurer les seuils de métrique du code pour l’analyse. Les règles suivantes sont configurables dans ce fichier :

- [CA1501 : Éviter l'excès d'héritage](ca1501.md)
- [CA1502 : Éviter l'excès de complexité](ca1502.md)
- [CA1505 : Éviter le code impossible à maintenir](ca1505.md)
- [CA1506 : Éviter les couplages de classe excessifs](ca1506.md)

Ce fichier de configuration s’attend à ce que chaque entrée soit au format suivant :

```ini
'RuleId'(Optional 'SymbolKind'): 'Threshold'
```

- Les valeurs valides pour « RuleId `CA1501`» `CA1502`sont `CA1505` , `CA1506`et.
- Les valeurs valides pour « SymbolKind » `Assembly`facultatives `Type`sont `Method`, `Field` `Namespace`, `Event`,, `Property`, et.
- Les valeurs valides pour « threshold » sont des entiers non négatifs.
- Les lignes commençant par' # 'sont traitées comme des lignes de commentaire

Par exemple, voici un fichier de configuration valide :

```ini
# Comment text

CA1501: 1

CA1502(Type): 4
CA1502(Method): 2
```

Une entrée non valide dans ce fichier de configuration est marquée avec `CA1509` le diagnostic.

> [!NOTE]
> La règle ca1509 n’est pas disponible dans l’analyse héritée. Il a été introduit pour la première fois dans les [analyseurs FxCop](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers) version 2.9.6.


## <a name="how-to-fix-violations"></a>Comment corriger les violations

Pour corriger une violation de cette règle, assurez-vous que l' `CodeMetricsConfig.txt` entrée non valide dans obtient le format requis.

## <a name="when-to-suppress-warnings"></a>Quand supprimer les avertissements

Ne supprimez pas les violations de cette règle.
