# <a name="c6230"></a>C6230

avertissement C6230 : cast implicite entre types d’entiers sémantiquement différents : utilisation de HRESULT dans un contexte booléen

Cet avertissement indique qu’un Bare `HRESULT` est utilisé dans un contexte où un résultat booléen est attendu, par exemple une instruction `if`. Ce test est susceptible de produire des résultats incorrects. Par exemple, la valeur de succès typique pour `HRESULT` (`S_OK`) est false lorsqu’elle est testée en tant que valeur booléenne.

## <a name="example"></a>Exemple

Le code suivant génère cet avertissement :

```cpp
#include <windows.h>

VOID f( )
{
  LPMALLOC pMalloc;
  HRESULT hr = CoGetMalloc(1, &pMalloc);

  if (hr)
  {

    // code ...
  }
  else
  {
    // code ...
  }
}
```

Dans la plupart des cas, la macro [Succeeded](/windows/desktop/api/winerror/nf-winerror-succeeded) ou [failed](/windows/desktop/api/winerror/nf-winerror-failed) doit être utilisée pour tester la valeur de l' `HRESULT`. Pour corriger cet avertissement, utilisez le code suivant :

```cpp
#include <windows.h>

VOID f( )
{
  LPMALLOC pMalloc;
  HRESULT hr = CoGetMalloc(1, &pMalloc);

  if (SUCCEEDED(hr))
  {

    // code ...
  }
  else
  {
    // code ...
  }
}
```

Pour cet avertissement, le type `SCODE` est traité comme une `HRESULT`.

L’utilisation de `malloc` et `free` (et les API de mémoire dynamique associées) présente de nombreux pièges comme une cause de fuites de mémoire et d’exceptions. Pour éviter ces types de fuites et de problèmes d’exception, utilisez les classes de pointeur et de C++ conteneur fournies par la bibliothèque standard. Il s’agit notamment de [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)et [Vector](/cpp/standard-library/vector). Pour plus d’informations, consultez [pointeurs intelligents](/cpp/cpp/smart-pointers-modern-cpp) et [ C++ bibliothèque standard](/cpp/standard-library/cpp-standard-library-reference).
