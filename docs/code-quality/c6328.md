# <a name="c6328"></a>C6328

Avertissement C6328 : Incompatibilité de taille : \<Type > passé comme paramètre \<number > quand \<Type > est requis dans l’appel à \<function >

Le passage d’un argument de type `char` au runtime C, les routines basées sur des caractères nommées `is<xxx>()`, par exemple, `isspace()`, peuvent avoir des résultats imprévisibles. Par exemple, un caractère codé sur un octet SBCS ou MBCS de type `char` avec une valeur supérieure à `0x7F` est une valeur négative. Si un `char` est passé, le compilateur peut convertir la valeur en `int` signé ou signé `long`. Cette valeur peut être étendue par le compilateur, avec des résultats inattendus. Par exemple, `isspace` accepte un argument de type `int` ; Toutefois, la plage de valeurs valide pour son argument d’entrée est :

`0 <= c <= 255`, plus la valeur spéciale `EOF`.

## <a name="example"></a>Exemple

Par défaut, `char` est un type signé en Visual C++. par conséquent, la plage de valeurs d’une variable de type char est `-128 <= c <= 127`. Par conséquent, si vous avez effectué les opérations suivantes, `c` serait signé avec une `int` signée avec la valeur-37, qui est en dehors de la plage valide pour [isspace](/cpp/standard-library/locale-functions#isspace):

```cpp
#include <iostream>

void f( )
{
    char c = -37;
    int retVal = isspace( c );
    // code ...
}
```

Pour corriger ce problème, vous pouvez utiliser `static_cast`, comme indiqué dans le code suivant :

```cpp
#include <iostream>

void f( )
{
    char c = -37;
    int retVal = isspace( static_cast<unsigned char> (c) );
    // code ...
}
```

L’avertissement C6328 existe spécifiquement pour intercepter ce bogue. Pour les caractères dans la plage ASCII de 7 bits, le cast n’est pas nécessaire. Les caractères en dehors de cette plage peuvent avoir des résultats imprévisibles, par exemple, une erreur de programme et un arrêt.
