# <a name="ca1829-use-lengthcount-property-instead-of-enumerablecount-method"></a>CA1829 : utilisez la propriété Length/Count à la place de la méthode Enumerable. Count

|||
|-|-|
|TypeName|UsePropertyInsteadOfCountMethodWhenAvailableAnalyzer|
|CheckId|CA1829|
|Category|Microsoft. performance|
|Modification avec rupture|Sans rupture|

## <a name="cause"></a>Cause

La <xref:System.Linq.Enumerable.Count%2A> méthode LINQ a été utilisée sur un type qui prend en charge une propriété `Length` équivalente, plus efficace ou `Count` .

## <a name="rule-description"></a>Description de la règle

Cette règle signale <xref:System.Linq.Enumerable.Count%2A> les appels de méthode LINQ sur les collections de types qui ont une propriété `Length` équivalente, mais plus efficace ou `Count` pour extraire les mêmes données. `Length`ou `Count` la propriété n’énumère pas la collection, par conséquent, est plus efficace.

Cette règle signale <xref:System.Linq.Enumerable.Count%2A> les appels sur les types de `Length` collection suivants avec la propriété :

- <xref:System.Array?displayProperty=fullName>
- <xref:System.Collections.Immutable.ImmutableArray%601?displayProperty=fullName>

Cette règle signale <xref:System.Linq.Enumerable.Count%2A> les appels sur les types de collections `Count` suivants à l’aide de la propriété :

- <xref:System.Collections.ICollection?displayProperty=fullName>
- <xref:System.Collections.Generic.ICollection%601?displayProperty=fullName>
- <xref:System.Collections.Generic.IReadOnlyCollection%601?displayProperty=fullName>

Les types de collection analysés peuvent être étendus à l’avenir pour couvrir plus de cas.

> [!NOTE]
> La règle CA1829 n’est pas disponible dans l’analyse héritée. Il a été introduit pour la première fois dans les [analyseurs FxCop](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers) version 2.9.6.

## <a name="how-to-fix-violations"></a>Comment corriger les violations

Pour corriger une violation, remplacez l' <xref:System.Linq.Enumerable.Count%2A> appel de méthode par l’utilisation `Length` de `Count` l’accès à la propriété ou. Par exemple, les deux extraits de code suivants illustrent une violation de la règle et comment la corriger :

```csharp
using System.Collections.Generic;
using System.Linq;

class C
{
    public int GetCount(int[] array)
        => array.Count();

    public int GetCount(ICollection<int> collection)
        => collection.Count();
}
```


```csharp
using System.Collections.Generic;

class C
{
    public int GetCount(int[] array)
        => array.Length;

    public int GetCount(ICollection<int> collection)
        => collection.Count;
}
```

> [!TIP]
> Un correctif de code est disponible pour cette règle dans Visual Studio. Pour l’utiliser, positionnez le curseur sur la violation et appuyez sur **CTRL**+**.** (point). Choisissez **utiliser la propriété longueur/nombre au lieu de Count () lorsqu'** elle est disponible dans la liste des options présentées.
>
> ![Correction de code pour CA1829-utilisez la propriété Length/Count au lieu de Count () quand elle est disponible](media/ca1829-codefix.png)

## <a name="when-to-suppress-warnings"></a>Quand supprimer les avertissements

Il est possible de supprimer en toute sécurité une violation de cette règle si vous ne vous inquiétez pas de l’impact sur les performances d’une énumération de collection inutile pour calculer le nombre.
