# <a name="ca1067-override-equals-when-implementing-iequatable"></a>CA1067: Remplacer Equals lors de l’implémentation d’IEquatable

|||
|-|-|
|TypeName|EquatableAnalyzer|
|CheckId|Ca1067|
|Category|Microsoft. Design|
|Modification avec rupture|Sans rupture|

## <a name="cause"></a>Cause

Un type implémente <xref:System.IEquatable%601>, mais ne se substitue <xref:System.Object.Equals%2A> pas à la méthode.

## <a name="rule-description"></a>Description de la règle

Un type qui <xref:System.IEquatable%601> implémente l’interface indique qu’il prend en charge la comparaison d’égalité de deux instances du type. Vous devez également substituer les implémentations de la classe <xref:System.Object.Equals%2A> de <xref:System.Object.GetHashCode> base des méthodes et afin que leur comportement soit cohérent avec <xref:System.IEquatable%601.Equals%2A?displayProperty=fullName> celui de l’implémentation. Consultez ce [document](/dotnet/api/system.iequatable-1#notes-to-implementers) pour plus d’informations.

Votre <xref:System.Object.Equals%2A> implémentation de doit retourner des résultats cohérents <xref:System.IEquatable%601.Equals%2A?displayProperty=fullName> avec l’implémentation.

## <a name="how-to-fix-violations"></a>Comment corriger les violations

Pour corriger une violation, substituez <xref:System.Object.Equals%2A> -la et implémentez-la <xref:System.IEquatable%601.Equals%2A?displayProperty=fullName> en appelant l’implémentation. Par exemple, les deux extraits de code suivants illustrent une violation de la règle et comment la corriger :

```csharp
using System;

public struct S : IEquatable<S>
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public bool Equals(S other)
        => _value == other._value;
}
```

```csharp
using System;

public struct S : IEquatable<S>
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public bool Equals(S other)
        => _value == other._value;

    public override bool Equals(object obj)
        => obj is S objS && Equals(objS);

    public override int GetHashCode()
        => _value.GetHashCode();
}
```

## <a name="when-to-suppress-warnings"></a>Quand supprimer les avertissements

Ne supprimez pas les violations de cette règle.
