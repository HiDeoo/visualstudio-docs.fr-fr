# <a name="idebugthread2cansetnextstatement"></a>IDebugThread2::CanSetNextStatement
Détermine si le pointeur d’instruction actuel peut être réglé sur le cadre de pile donné.

## <a name="syntax"></a>Syntaxe

```cpp
HRESULT CanSetNextStatement ( 
   IDebugStackFrame2*  pStackFrame,
   IDebugCodeContext2* pCodeContext
);
```

```csharp
int CanSetNextStatement ( 
   IDebugStackFrame2  pStackFrame,
   IDebugCodeContext2 pCodeContext
);
```

## <a name="parameters"></a>Paramètres
`pStackFrame`\
Réservé à une utilisation future; d’une valeur nulle. S’il s’agit d’une valeur nulle, utilisez le cadre de pile actuel.

`pCodeContext`\
[dans] Un objet [IDebugCodeContext2](../../../extensibility/debugger/reference/idebugcodecontext2.md) qui décrit l’emplacement du code sur le point d’être exécuté et son contexte.

## <a name="return-value"></a>Valeur de retour
 En cas de réussite, retourne `S_OK` , sinon, retourne un code d'erreur.

## <a name="remarks"></a>Notes
 Si cette `S_OK`méthode revient, appelez alors la méthode [SetNextStatement](../../../extensibility/debugger/reference/idebugthread2-setnextstatement.md) pour définir réellement la prochaine déclaration.
