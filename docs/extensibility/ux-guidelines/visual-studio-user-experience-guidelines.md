# <a name="visual-studio-user-experience-guidelines"></a>Lignes directrices pour l’expérience utilisateur dans Visual Studio
Suivez les directives d’expérience utilisateur visual studio si vous concevez de nouvelles fonctionnalités pour Visual Studio. Ces lignes directrices contiennent des informations sur les modèles d’utilisateur et les modèles d’interaction courants afin que les concepteurs de nouvelles interfaces utilisateur (interfaces utilisateur) puissent créer des expériences utilisateur transparentes et cohérentes dans l’environnement de développement.

Si vous développez des logiciels pour les produits Microsoft, apprenez à utiliser ces ressources :

- Les [lignes directrices sur l’interaction avec l’expérience utilisateur Windows](/windows/win32/uxguide/guidelines) sont les bases du comportement et de l’apparence de bureau Windows.

- Ces lignes directrices Visual Studio, qui sont des lignes directrices spécifiques à la plate-forme qui remplacent les directives générales de Windows, le cas échéant, dans notre environnement.

- Les [contrôles et les modèles pour les applications UWP](/windows/uwp/design/controls-and-patterns) sont une référence pour les modèles émergents qui peuvent être utilisés dans certains cas au sein de Visual Studio.

Ces lignes directrices vous donnent des informations sur ce qu’il faut faire lors de la conception de Visual Studio, et dans certains cas des informations sur la façon de le faire est également inclus. Pour des informations de mise en œuvre plus détaillées et pour en savoir plus sur des interfaces de programmation d’applications spécifiques (API) qui vous permettent d’écrire du code pour les solutions qui s’intègrent dans l’environnement de développement intégré Visual Studio (IDE), utilisez le [Kit de développement logiciel Visual Studio (SDK).](../visual-studio-sdk.md)
