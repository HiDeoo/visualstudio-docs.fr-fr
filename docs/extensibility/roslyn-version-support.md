# <a name="net-compiler-platform-package-version-reference"></a>.NET compiler plate-forme de référence de version paquet

Le tableau suivant montre quelles versions [de la plate-forme de compilateur .NET (Roslyn)](https://www.nuget.org/packages/Microsoft.Net.Compilers/) sont prises en charge pour différentes versions de Visual Studio.

Par exemple, pour s’assurer que votre analyseur personnalisé fonctionne sur toutes les versions de Visual Studio 2017, il devrait cibler la version 2.0 de Microsoft.Net.Compilers.

| Version du paquet Roslyn | Version Visual Studio prise en charge minimale |
| - | - |
| 3.x | Visual Studio 2019 |
| 2.10.0 | Visual Studio 2017 version 15.9 |
| 2.9.0 | Visual Studio 2017 version 15.8 |
| 2.8.2 | Visual Studio 2017 version 15.7 |
| 2.7.0 | Visual Studio 2017 version 15.6 |
| 2.6.1 | Visual Studio 2017 version 15.5 |
| 2.4.0 | Visual Studio 2017 version 15.4 |
| 2.3.2 | Visual Studio 2017 version 15.3 |
| 2.2.0 | Visual Studio 2017 version 15.2 |
| 2.1.0 | Visual Studio 2017 version 15.1 |
| 2.0.0 | Studio visuel 2017 RTM |
| 1.3.2 | Visual Studio 2015 mise à jour 3 |
| 1.2.2 | Visual Studio 2015 mise à jour 2 |
| 1.1.1 | Visual Studio 2015 mise à jour 1 |
| 1.0.1 | Studio visuel 2015 RTM |

> [!TIP]
> Pour les forfaits Roslyn où la version Visual Studio minimale est une version Visual Studio 2017, toutes les versions de Visual Studio 2019 sont également prises en charge parce qu’elles sont venues plus tard.
