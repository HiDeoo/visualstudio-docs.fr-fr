# <a name="modifying-the-isolated-shell-by-using-the-pkgdef-file"></a>Modification du shell isolé à l’aide du fichier .Pkgdef
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Le fichier. pkgdef prend en charge les paramètres que vous pouvez utiliser pour personnaliser une application de Shell isolé. Elle spécifie les valeurs qui sont créées lors de l’installation d’une application sur un ordinateur et qui sont référencées par le shell Visual Studio au démarrage de l’application. Les paramètres sont organisés dans le fichier en fonction des clés de Registre applicables.  

> [!WARNING]
> Notez que les fichiers. pkgdef qui ne sont pas déclarés dans le fichier. vsixmanifest du VSPackage ne sont pas analysés au démarrage de Visual Studio.  

 Le fichier. pkgdef contient des sections qui sont identifiées par une clé, `[$RootKey$]` ou une `[$RootKey$\` *sous-* clé `]` , où $ROOTKEY $ est la clé racine de l’application.  

 Chaque section contient des paires nom/valeur qui ont le format suivant : `"` *ValueName* `"=` *valeur*valueName.  

 Les valeurs sont soit une chaîne placée entre guillemets, soit un entier 32 bits représenté sous forme de valeur DWORD. Les valeurs ont les contraintes suivantes :  

- Toutes les valeurs DWORD sont au format hexadécimal, par exemple `dword:00000001` .  

   Pour les valeurs booléennes, 1 représente la valeur true et 0 représente la valeur false.  

- Toutes les chaînes GUID sont au format de Registre, par exemple, `"{00000000-0000-0000-0000-000000000000}"` .  

- Tous les identificateurs de ressource localisables se présentent sous la forme « @*ResourceId*» ou « #*ResourceId*», où *ResourceId* est l’identificateur de ressource dans le package d’interface utilisateur de l’application, par exemple, `"@102"` . Le package de l’interface utilisateur de l’application est le package référencé dans le paramètre AppLocalizationPackage.  

  Par exemple,  

```  
"HideSolutionConcept"=dword:00000001  
"DefaultDebugEngine"="{00000000-0000-0000-0000-000000000000}"  
```  

 Vous pouvez ajouter des commentaires au fichier. pkgdef. Un commentaire sur une seule ligne a deux barres obliques comme deux premiers caractères.  

 Pour obtenir la liste des chaînes de substitution, consultez [chaînes de substitution utilisées dans. Pkgdef et. Fichiers Pkgundef](../extensibility/substitution-strings-used-in-dot-pkgdef-and-dot-pkgundef-files.md).  

 Les sections suivantes décrivent des valeurs de Registre spécifiques qui affectent le comportement du shell Visual Studio en mode isolé. Vous pouvez également définir des valeurs de Registre supplémentaires pour l’application dans ce fichier.  

> [!NOTE]
> Si un paramètre n’est pas fourni dans le fichier. pkgdef, aucune entrée correspondante n’est effectuée dans le registre.

## <a name="binding-path-settings"></a>Paramètres de chemin d’accès de liaison  
 La clé [$RootKey $ \BindingPaths \\ {00000000-0000-0000-0000-000000000000} ] contient la liste des répertoires dans lesquels l’interpréteur de commandes recherche les assemblys. Ces répertoires sont ajoutés à la liste des répertoires que le shell sonde pour les assemblys privés pour l’application.  

 Par défaut, aucune entrée de chemin d’accès de liaison n’est ajoutée au fichier. pkgdef. Toutefois, les sous-répertoires suivants du répertoire d’installation de Visual Studio sont automatiquement ajoutés à la liste Binding-Path de l’application dans le registre.  

- Common7\IDE\  

- Common7\IDE \\ \PrivateAssemblies  

- Common7\IDE \\ \PublicAssemblies  

  Pour ajouter un répertoire au chemin d’accès de liaison, ajoutez une entrée sous la forme «*DirectoryName*» = «», où *DirectoryName* est un chemin d’accès absolu. Par exemple,  

```  
[$RootKey$\BindingPaths\{00000000-0000-0000-0000-000000000000}]  
"$RootFolder$\directory1"=""  
"%CommonProgramFiles%\directory2"=""  
```

## <a name="package-menu-item-settings"></a>Paramètres de l’élément de menu package  
 La clé de registre [$RootKey $ \Menus] définit les fichiers de ressources d’interface utilisateur pour l’application.  

 Les valeurs d’élément de menu se présentent sous la forme "{*vsUiPackageGuid*}" = ", *ResourceId*, *versionNumber*", où *vsUiPackageGuid* est le GUID du package d’interface utilisateur de l’application, *ResourceId* est l’identificateur de ressource de la ressource CTMENU qui contient les éléments d’interface utilisateur et *versionNumber* est un numéro de version virtuelle pour la ressource CTMENU. Pour plus d’informations, consultez [inscription des gestionnaires de commandes d’assembly d’interopérabilité](../extensibility/internals/registering-interop-assembly-command-handlers.md).  

 Par défaut, une entrée d’élément de menu est créée dans le fichier. pkgdef pour le package d’interface utilisateur de l’application.  

 Pour chaque package qui fournit des éléments de menu et qui est distribué dans le cadre de l’application, ajoutez une entrée d’élément de menu pour le package.
