# <a name="visual-studio-user-experience-guidelines"></a>Lignes directrices pour l’expérience utilisateur dans Visual Studio
[!INCLUDE[vs2017banner](../../includes/vs2017banner.md)]

Les instructions relatives à l’expérience utilisateur de Visual Studio sont destinées aux personnes qui conçoivent de nouvelles fonctionnalités pour Visual Studio. Ces instructions contiennent des informations sur les modèles utilisateur communs et les modèles d’interaction afin que les concepteurs de nouvelles interfaces utilisateur puissent créer des expériences utilisateur qui sont transparents et cohérentes dans l’environnement de développement.  
  
 Le développement de logiciels pour les produits Microsoft consiste à comprendre les instructions fournies par Windows. Il existe trois ressources importantes à connaître :  
  
- [Instructions relatives à l’interaction avec l’expérience utilisateur Windows](https://msdn.microsoft.com/library/aa511258.aspx). Ces instructions sont les principes de base du comportement et de l’apparence des postes de travail Windows.  
  
- Ces instructions de Visual Studio, qui sont des instructions spécifiques à la plateforme qui remplacent les instructions Windows générales, là où cela s’avère approprié pour notre environnement.  
  
- Les [contrôles et les modèles pour les applications UWP](/windows/uwp/design/controls-and-patterns) sont une référence pour les modèles émergents qui peuvent être utilisés dans certaines instances de Visual Studio.  
  
  Ces instructions vous donnent des informations sur la procédure à suivre lors de la conception de Visual Studio et, dans certains cas, des informations sur la façon de procéder sont également incluses. Pour plus d’informations sur l’implémentation et pour en savoir plus sur les interfaces de programmation d’applications (API) spécifiques qui vous permettent d’écrire du code pour les solutions qui s’intègrent dans l’environnement de développement intégré (IDE) de Visual Studio, utilisez le [Kit de développement logiciel (SDK) de Visual Studio](https://msdn.microsoft.com/library/bb166441.aspx).
