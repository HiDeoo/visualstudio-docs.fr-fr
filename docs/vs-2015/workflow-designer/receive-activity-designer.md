# <a name="receive-activity-designer"></a>Concepteur d'activités Receive
Le concepteur d’activités **Receive** est <xref:System.ServiceModel.Activities.Receive> utilisé pour créer et configurer une activité. Une activité <xref:System.ServiceModel.Activities.Receive> est une activité qui reçoit un message qui peut être un type intégré, tel que <xref:System.ServiceModel.Channels.Message>, <xref:System.IO.Stream> ou <xref:System.Xml.Linq.XElement>, ou un contrat de données défini par l'application, un contrat de message ou une classe XML sérialisable.

## <a name="the-receive-activity"></a>Activité Receive
 L'activité <xref:System.ServiceModel.Activities.Receive> peut recevoir un élément unique ou plusieurs éléments selon le type de contenu de réception utilisé. Une activité <xref:System.ServiceModel.Activities.SendReply> peut être liée à une activité <xref:System.ServiceModel.Activities.Receive> qui reçoit un message dans le cadre d’un modèle d’échange de messages de demande/réponse sur le service.

### <a name="using-the-receive-activity-designer"></a>Utilisation du concepteur d'activités Receive
 Le concepteur d’activités **Receive** se trouve dans la catégorie **Messagerie** de la Boîte à [!INCLUDE[wfd2](../includes/wfd2-md.md)] **outils**, qui est accessible en cliquant sur l’onglet **Toolbox** sur le (Alternativement, sélectionnez barre d’outils du menu **View** ou CTRL-ALT-X.) **Toolbar**

 Le concepteur d’activités **Receive** peut être traîné de [!INCLUDE[wfd2](../includes/wfd2-md.md)] la boîte à **outils** et laissé tomber à la surface partout où les activités sont généralement placées. Cette opération crée une activité <xref:System.ServiceModel.Activities.Receive> avec une propriété <xref:System.Activities.Activity.DisplayName%2A> affectée de la valeur par défaut Receive. Le <xref:System.Activities.Activity.DisplayName%2A> peut être édité dans l’en-tête du concepteur d’activité **Receive** ou dans la boîte **DisplayName** de la grille de propriété.

 Pour créer <xref:System.ServiceModel.Activities.SendReply> une activité et <xref:System.ServiceModel.Activities.Receive> la lier à l’activité sélectionnée, cliquez à droite sur le concepteur d’activités **Recevoir,** cliquez sur l’élément **Create SendReply** dans le menu contextuelle et le concepteur **SendReplyToReceive** apparaît ci-dessous le concepteur **Receive.** L’activité <xref:System.ServiceModel.Activities.SendReply> est une activité qui envoie le message de réponse dans le cadre d’un modèle d’échange de messages de demande/réponse sur le service. Il peut être configuré avec le concepteur **SendReplyToReceive.**

 Alternativement, le concepteur de modèle **ReceiveAndSendReply** dans la catégorie **Messagerie** de la boîte <xref:System.ServiceModel.Activities.Receive> <xref:System.ServiceModel.Activities.SendReply> à **outils** peut être utilisé pour créer une paire de préconfiguré et d’activité. [!INCLUDE[crabout](../includes/crabout-md.md)]l’utilisation du modèle **ReceiveAndSendReply** et **SendReplyToReceive,** voir le sujet [ReceiveAndSendReply.](../workflow-designer/receiveandsendreply-template-designer.md)
