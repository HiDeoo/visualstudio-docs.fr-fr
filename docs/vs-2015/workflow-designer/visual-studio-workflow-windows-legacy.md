# <a name="visual-studio-workflow-windows-legacy"></a>Visual Studio pour Windows Workflow (hérité)
Le [!INCLUDE[wfd2](../includes/wfd2-md.md)] hérité fourni par [!INCLUDE[vs2010](../includes/vs2010-md.md)] peut être utilisé pour cibler le [!INCLUDE[netfx35_long](../includes/netfx35-long-md.md)] ou le [!INCLUDE[vstecwinfx](../includes/vstecwinfx-md.md)]. Le concepteur hérité est un concepteur visuel que vous pouvez utiliser pour créer un workflow. Vous pouvez faire glisser des activités de la **boîte à outils** vers l’aire de conception, puis les configurer ici. En tant que fenêtre d'éditeur [!INCLUDE[vsprvs](../includes/vsprvs-md.md)], elle occupe la zone de fenêtre principale utilisée par d'autres fenêtres d'éditeur [!INCLUDE[vsprvs](../includes/vsprvs-md.md)]. Le nom du workflow est affiché dans la barre de titre de la fenêtre [!INCLUDE[vsprvs](../includes/vsprvs-md.md)].

 Tout utilisateur [!INCLUDE[vsprvs](../includes/vsprvs-md.md)] expérimenté a l'habitude de travailler avec la plupart des fenêtres du concepteur hérité. Le tableau suivant décrit les fenêtres principales.

|Fenêtre|Description|
|------------|-----------------|
|Aire de conception de workflow héritée|Aire visuelle du concepteur utilisée pour créer un workflow qui cible le [!INCLUDE[netfx35_long](../includes/netfx35-long-md.md)] ou le [!INCLUDE[vstecwinfx](../includes/vstecwinfx-md.md)]. Il offre la possibilité de composer visuellement des activités en faisant glisser des concepteurs d’activités de la **boîte à outils** vers l’aire de conception.|
|Boîte à outils|Contient des représentations graphiques des activités, que vous pouvez faire glisser vers l'aire de conception.|
|Propriétés|Utilisée pour configurer les propriétés de l'activité de workflow.|
|l'Explorateur de solutions|Affiche des projets de workflow dans une arborescence hiérarchique et active l'accès aux fonctionnalités qui aident à gérer les projets.|
|Fenêtres de débogage|Les fenêtres de débogage Visual Studio standard, telles que les fenêtres **Threads**, **pile des appels**, point d' **arrêt**, **paramètres locaux**et **outil espion** .|
