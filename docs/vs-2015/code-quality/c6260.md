# <a name="c6260"></a>C6260
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

AVERTISSEMENT C6260 : sizeof * sizeof est presque toujours incorrect. voulez-vous utiliser un nombre de caractères ou un nombre d’octets ?  
  
 Cet avertissement indique que les résultats de deux opérations de `sizeof` ont été multipliés ensemble. L’opérateur CC++ /`sizeof` retourne le nombre d’octets de stockage qu’un objet utilise. Il est généralement incorrect de le multiplier par une autre opération de `sizeof` ; en général, il s’agit du nombre d’octets dans un objet ou du nombre d’éléments dans un tableau (par exemple, le nombre de caractères larges dans un tableau).  
  
 Un comportement non intuitif est associé à `sizeof` opérateur. Par exemple, en C, le `sizeof ('\0') == 4,`, car un caractère est d’un type intégral. Dans C++, le type d’un littéral de caractère est `char`, donc `sizeof ('\0') == 1`. Toutefois, dans C et C++, les conditions suivantes sont vraies :  
  
```  
sizeof ("\0") == 2.   
```  
  
## <a name="example"></a>Exemple  
 Le code suivant génère cet avertissement :  
  
```  
#include <windows.h>  
  
void f( )  
{  
  int i;  
  i = sizeof (L"String") * sizeof (WCHAR);  
  // code ...  
}  
```  
  
 Pour corriger cet avertissement, utilisez le code suivant :  
  
```  
#include <windows.h>  
  
void f( )  
{  
  int i;  
  i= sizeof (L"String") / sizeof (WCHAR);  
  
  /* or to get bytes */  
  i = sizeof (L"String");  
  // code ...  
}  
```
